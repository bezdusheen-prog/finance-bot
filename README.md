# 💰 Finance Bot

Telegram-бот для учёта личных финансов на aiogram 3.

## ✨ Возможности

- **Быстрый ввод** операций одной строкой
- **Автоматическое распознавание** категорий
- **Бюджетные конверты** с распределением зарплаты
- **Накопительные фонды** для целевых накоплений
- **Отчёты** за день/неделю/месяц
- **История операций** с удалением
- **Ежедневные напоминания** в 21:00
- **SQLite/PostgreSQL** поддержка
- - **Inline-клавиатура** для быстрого выбора категорий
- **Команда /quick** для быстрого добавления расхода одной строкой
- **Команда /summary** с детальной сводкой за день/неделю/месяц и топ-5 категорий
- **Автоматические уведомления** о превышении бюджета

## 🚀 Быстрый старт

### 1. Клонируйте репозиторий

```bash
git clone https://github.com/bezdusheen-prog/finance-bot.git
cd finance-bot
```

### 2. Установите зависимости

```bash
pip install -r requirements.txt
```

### 3. Настройте окружение

Создайте файл `.env`:

```bash
cp .env.example .env
```

Отредактируйте `.env` и укажите токен бота:

```
BOT_TOKEN=ваш_токен_от_BotFather
DB_URL=sqlite+aiosqlite:///finance_bot.db
TZ=Europe/Moscow
```

### 4. Запустите бота

```bash
python bot.py
```

## 📱 Примеры использования

**Быстрый ввод:**
```
350 кофе                    → расход 350 ₽ категория Развлечения
2200 продукты обед          → расход 2200 ₽ с комментарием
+120000 зарплата            → доход с распределением по конвертам
зарплата 120000             → альтернативный формат
```

**Команды:**
- `/start` — начало работы
- `/today` — отчёт за сегодня
- `/week` — отчёт за неделю
- `/month` — отчёт за месяц
- `/balance` — баланс по счетам
- `/history` — история операций
- `/categories` — список категорий
- `/budget` — бюджет и конверты
- `/funds` — накопительные фонды
- `/settings` — настройки
- `/del <id>` — удалить операцию

## 📦 Деплой на Bothost

1. Создайте бота на https://bothost.ru
2. Подключите GitHub репозиторий
3. Укажите переменные окружения:
   - `BOT_TOKEN`
   - `DB_URL` (опционально)
   - `TZ` (опционально)
4. Запустите деплой

## 🗄️ База данных

### Модели:
- **User** — пользователи
- **Account** — счета
- **Category** — категории доходов/расходов
- **Operation** — транзакции
- **Envelope** — бюджетные конверты
- **Fund** — накопительные фонды

### Миграция на PostgreSQL:

Измените `DB_URL` в `.env`:

```
DB_URL=postgresql+asyncpg://user:password@host:port/dbname
```
